让多选下拉框滚动到指定位置

最近做了一个用文本框(input[text])+下拉框(select) 实现的可编辑下拉框控件， 支持单选下拉框(select-one)和多选下拉框(select-multiple)。

下面我要说的是多选下拉框部分。

对于多选的下拉框，UI 还没有好的想法，目前是在下拉框之前加入一个文本框， 一个针对初级用户的按钮。

用户编辑的部分，处理方法如下：

  1. 比较整个列表项，如果已有则 option.selected = true，
  2. 否则 new Option(text, value, defaultSelected, selected);

然后需求方（其实是 PM）说，这个下拉框的数据量可能会很大。我想，可选的基数 太大，加上用户输入的部分可以使用特殊字符分隔一次选择/添加多个，对于内部系统 那些顽固的 IE6 用户岂不是要抱怨死？我干脆全部当作新增项好了，简单快捷。

呃，PM 又开始纠结了，“这种方式好不好”。 当然不好啦，需求本身就不好，做出来的东西会好么？

好吧这个先不说了，又说你插入到最后，用户最后看不到选中的，能不能插入到最前面， 或者滚动到最后？

插入到最前？要把上万个选项往后移，还不如做比较呢。

那就滚动到最后吧，用 scrollTop，但是发现 IE 在插入 new Option 时设置 scrollTop 是可能失效的，即使 setTimeout 也一样，难道要 setInterval，我们已经迷失了。

呃，就 scrollTop 这个问题本身来说，有解么？

语法:

new Option(text, value, defaultSelected, selected)

说明:

The Option constructor or the HTML tag. To create an Option object with its constructor:
Once you've created an Option object, you can add it to a selection list using the Select.optionsarray.

参数: text

(Optional) Specifies the text to display in the select list.

value

(Optional) Specifies a value that is returned to the server when the option is selected and the form is submitted.

defaultSelected

(Optional) Specifies whether the option is initially selected (true or false).

selected

(Optional) Specifies the current selection state of the option (true or false).
